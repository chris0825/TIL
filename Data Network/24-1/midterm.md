### HTTP 요청 메세지
> 메세지가 일반 ASCII 텍스트로 쓰여있어 사람이 읽을 수 있음<br>

|라인|줄번호|내용|
|:-:|:-:|:-|
|요청라인|1|방식 필드 + URL 필드 + HTTP 버전 필드|

|더|3|Connection: open/close ((비)지속 연결 여부)|
|라|4|User-agent: 내(클라이언트) 브라우저 정보|
|인|5|Accept-language: 클라이언트 선호 언어|
### 방식필드 종류
- GET
  > 브라우저가 URL 필드로 식별되는 객체를 요청할 때 사용
- POST
  > 사용자(클라이언트)의 입력이 필요할 때
  > ex) 회원가입, 장바구니
- HEAD
  > 웹 서버 정보만 필요할 때 (바디문 전송 X)
  > ex) 디버깅
- PUT
  > 웹 서버에 업로드 할 때
  > ex) 과제, 파일 업로드
- DELETE
  > 웹 서버에서 PUT으로 올린 객체를 삭제
### HTTP 응답 메세지
> 상태 라인: 프로토콜 '버전필드', '상태코드', '해당 상태 메세지'
- 상태코드 종류
  - 200 OK
    > 요청 성공
  - 301 Moved Permanently
    > URL이 변경되어 바뀐 URL로 안내
  - 304 Not modified
    > 바뀐게 없으므로 캐싱 복사복을 이용
  - 400 Bad Request
    > 서버가 클라이언트의 요청 메세지를 이해하지 못함
  - 404 Not Found
    > 요청 문서가 서버에 존재하지 않음
  - 505 HTTP Version Not Supported
    > 요청 HTTP 프로토콜 버전을 지원하지 않음
